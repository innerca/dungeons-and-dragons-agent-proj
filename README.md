# Dungeons & Dragons Agent Project

A DND game project powered by AI, built with a microservice architecture.

## Project Overview

This project creates an AI-driven Dungeons & Dragons experience where game logic is generated by LLM in real-time. The architecture follows a traditional game server pattern: **Gateway - GameServer - Database**.

## Architecture

```
Frontend (React+TS)  <--WS/SSE-->  Gateway (Go)  <--gRPC-->  GameServer (Python)  -->  LLM API
       :5173                          :8080                       :50051              (DeepSeek)
```

## Tech Stack

| Component | Technology |
|-----------|-----------|
| Frontend | React + TypeScript (Vite) |
| Gateway | Golang (chi router) |
| GameServer | Python + grpcio |
| Frontend Communication | WebSocket (send) + SSE (receive streaming) |
| Service Communication | gRPC (server streaming) |
| Database | PostgreSQL + Redis (future) |
| Containerization | Docker Compose |
| Python Package Manager | uv |
| LLM | DeepSeek (default), multi-model support |

## Security Principles

Security is the highest priority in this project, embedded at every architectural layer.

### Core Principles
1. **Defense in Depth**: Detection and interception at input, inference, memory, and execution layers
2. **Least Privilege**: Agents can only access the minimum tools and data needed to complete their tasks
3. **Human in the Loop**: Critical operations require human confirmation
4. **Observability**: All behaviors are traceable and reproducible

### Security Checklist

| Check Item | Method | Pass Criteria |
|-----------|--------|---------------|
| Prompt Injection Detection | Automated attacks via Garak, PromptInject | Attack success rate < 5% |
| Jailbreak Defense | Test 50+ known jailbreak templates | Rejection rate > 95% |
| Tool Parameter Validation | Abnormal types, SQL injection, path traversal | Tool refuses execution and logs |
| Permission Isolation | User A attempts to access User B's data | Returns unauthorized error |
| Output Leak Prevention | Random phone numbers, emails in replies | Desensitization module detects and masks |
| Cost Circuit Breaker | Simulate ultra-long conversations or loop calls | Terminates or escalates at threshold |
| Audit Completeness | Check logs for all critical events | Logs are complete and tamper-proof |

## Local Development

### Prerequisites
- Go 1.22+
- Python 3.9+
- Node.js 18+
- uv (Python package manager)
- protoc (Protocol Buffers compiler)
- Docker & Docker Compose (optional)

### Quick Start

```bash
# 1. Clone and setup
git clone https://github.com/innerca/dungeons-and-dragons-agent-proj.git
cd dungeons-and-dragons-agent-proj

# 2. Copy env file and fill in your API keys
cp .env.example .env
# Edit .env with your keys

# 3. Generate gRPC code
make proto-gen

# 4. Start services (choose one)
make dev              # Docker Compose (all services)
# OR start individually:
make dev-gameserver   # Python GameServer on :50051
make dev-gateway      # Go Gateway on :8080
make dev-frontend     # React Frontend on :5173
```
