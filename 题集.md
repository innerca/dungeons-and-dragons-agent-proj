## 伪代码作答的能力边界声明

本套题集允许候选人以逻辑严密的伪代码作答，旨在聚焦于**逻辑正确性、边界条件处理和状态流转设计**。

**伪代码形式天然无法有效评估以下能力**，这些维度将在后续面试环节中另行考察：

| 无法评估的维度 | 后续考察方式 |
| :--- | :--- |

**关于 L1 评级的补充说明**

- 核心逻辑存在明显漏洞（如并发竞态、资源泄漏）
- 边界条件处理严重缺失（如空值、超限无任何处理）
- 经提示后仍无法修正或理解问题所在

### 热身题 0：固定大小的循环队列

**场景**  
需要一个简单的内存缓冲区，存储最近 N 条日志或消息。请用数组实现一个固定容量的循环队列，支持 `enqueue(item)` 和 `dequeue()` 操作。当队列满时，`enqueue` 可以覆盖最旧的数据或返回错误（任选一种并说明）。

**要求**  
用你最熟悉的语言或清晰的伪代码写出核心逻辑。

**考察点**
- 数组索引管理（`head`、`tail` 指针）
- 队列空/满的判断条件
- 边界条件处理（初始状态、指针回绕）

**分级评估**

| 等级 | 表现 |
| :--- | :--- |
| L3 | 5 分钟内写出正确逻辑，指针更新无误，边界清晰。 |
| L2 | 能写出大致结构，但空/满判断有瑕疵，经提示修正。 |
| L1 | 思路混乱，无法写出可工作的逻辑。 |

**伪代码参考（Python 风格）**

```python
class CircularQueue:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.data = [None] * capacity
        self.head = 0
        self.tail = 0
        self.size = 0

    def enqueue(self, item):
        if self.size == self.capacity:
            self.head = (self.head + 1) % self.capacity
            self.size -= 1
        self.data[self.tail] = item
        self.tail = (self.tail + 1) % self.capacity
        self.size += 1

    def dequeue(self):
        if self.size == 0:
            return None
        item = self.data[self.head]
        self.head = (self.head + 1) % self.capacity
        self.size -= 1
        return item
```

**伪代码参考（Go 风格）**

```go
    data     []interface{}
    capacity int
    head     int
    tail     int
    size     int
}

func NewCircularQueue(capacity int) *CircularQueue {
    return &CircularQueue{
        data:     make([]interface{}, capacity),
        capacity: capacity,
    }
}

func (q *CircularQueue) Enqueue(item interface{}) {
    if q.size == q.capacity {
        q.head = (q.head + 1) % q.capacity
        q.size--
    }
    q.data[q.tail] = item
    q.tail = (q.tail + 1) % q.capacity
    q.size++
}

func (q *CircularQueue) Dequeue() interface{} {
    if q.size == 0 {
        return nil
    }
    item := q.data[q.head]
    q.head = (q.head + 1) % q.capacity
    q.size--
    return item
}
```

### 热身题 0A：验证字符串是否为有效数字

**场景**  
系统需要校验用户输入的字符串能否安全地转换为数值（如配置值、API 参数）。请实现一个函数，判断给定字符串是否表示一个有效的整数或浮点数（可包含负号和小数点，但不接受科学计数法或其他格式）。

**要求**  
写出核心判断逻辑，不要求正则表达式，用遍历字符的方式或语言内置转换均可，但需处理边界。

可假设输入字符串仅包含：数字 `0-9`、负号 `-`、小数点 `.`。有效的数字格式定义如下：
- 负号（如有）必须出现在首位
- 小数点最多出现一次，且不能作为唯一字符（如 `"."` 无效）
- 至少包含一位数字（如 `"-"` 无效）

**考察点**
- 空字符串、纯符号、多个小数点等边界处理
- 代码的防御性
- 是否考虑语言内置函数的局限性

**分级评估**

| 等级 | 表现 |
| :--- | :--- |
| L3 | 正确处理空串、正负号、多个小数点、非数字字符。 |
| L2 | 能写出基本逻辑，遗漏一两种边界。 |
| L1 | 逻辑漏洞明显，如认为 `"1.2.3"` 有效。 |

**伪代码参考（Python 风格）**

```python
def is_valid_number(s: str) -> bool:
    if not s:
        return False
    
    seen_dot = False
    seen_digit = False
    start = 0
    
    if s[0] == '-':
        if len(s) == 1:
            return False
        start = 1
    
    for i in range(start, len(s)):
        c = s[i]
        if c == '.':
            if seen_dot:
                return False
            seen_dot = True
        elif '0' <= c <= '9':
            seen_digit = True
        else:
            return False
    
    return seen_digit
```

### 热身题 0B：带过期时间的简单键值对

**场景**  
实现一个极简的内存缓存，支持 `set(key, value, ttl_seconds)` 和 `get(key)`。不需要考虑并发或内存淘汰，只需在 `get` 时检查是否过期。

**要求**  
写出核心数据结构及两个方法的逻辑。

**考察点**
- 基础数据结构（字典/Map）
- 时间戳获取与比较
- 过期后的清理策略（惰性删除）

**分级评估**

| 等级 | 表现 |
| :--- | :--- |
| L3 | 正确实现 set/get，惰性删除逻辑清晰。 |
| L2 | 能写出基本的数据结构，但在 `get` 方法的过期判断逻辑上有遗漏（如只判断存在性而忽略时间比较），经提示后能修正。 |
| L1 | 无法组织数据结构或时间逻辑混乱。 |

**伪代码参考（Python 风格）**

```python
import time
from typing import Any, Dict, Optional

class SimpleCache:
    def __init__(self):
        self.data: Dict[str, tuple[Any, float]] = {}
    
    def set(self, key: str, value: Any, ttl_seconds: int):
        expire_at = time.time() + ttl_seconds
        self.data[key] = (value, expire_at)
    
    def get(self, key: str) -> Optional[Any]:
        entry = self.data.get(key)
        if entry is None:
            return None
        value, expire_at = entry
        if expire_at < time.time():
            del self.data[key]
            return None
        return value
```

**伪代码参考（Go 风格）**

```go
import "time"

type cacheEntry struct {
    value    interface{}
    expireAt time.Time
}

type SimpleCache struct {
    data map[string]cacheEntry
}

func NewSimpleCache() *SimpleCache {
    return &SimpleCache{
        data: make(map[string]cacheEntry),
    }
}

func (c *SimpleCache) Set(key string, value interface{}, ttlSeconds int) {
    expireAt := time.Now().Add(time.Duration(ttlSeconds) * time.Second)
    c.data[key] = cacheEntry{value: value, expireAt: expireAt}
}

func (c *SimpleCache) Get(key string) interface{} {
    entry, ok := c.data[key]
    if !ok {
        return nil
    }
    if time.Now().After(entry.expireAt) {
        delete(c.data, key)
        return nil
    }
    return entry.value
}
```

### 题 1：带 TTL 的请求去重缓存

**场景**  
同一 `request_id` 可能因网络重试被多次发送到 Agent 系统。需要保证同一 `request_id` 的实际业务逻辑只被执行一次，且缓存条目在 5 分钟后自动过期，防止内存无限增长。

**要求**  
实现 `IdempotentCache` 组件的 `get_or_execute(request_id, fn)` 方法。

**考察点**
- 并发安全：多请求同时到达时的竞态处理
- TTL 设计：过期机制的实现思路（惰性删除或定时清理）
- 结果缓存：区分成功与失败结果的缓存策略
- 边界条件：`request_id` 为空、执行函数抛异常时的处理

**分级评估**

| 等级 | 表现 |
| :--- | :--- |
| L4 | 完整描述锁的粒度（读写锁分离）；讨论 TTL 惰性删除与后台清理的取舍；失败结果缓存且 TTL 更短；考虑内存上限。 |
| L3 | 正确使用互斥锁保护临界区；TTL 用时间戳+检查实现；考虑了异常时清理 in-flight 状态。 |
| L2 | 能描述基本思路，但忽略并发竞态或 TTL 描述模糊。经提示后能修正。 |
| L1 | 无法描述并发控制；认为“一个 Map 就够了”；对 TTL 无有效思路。 |

**伪代码参考（Python 风格）**

```python
import time
import threading
from typing import Any, Callable, Dict, Optional
from concurrent.futures import Future

class IdempotentCache:
    def __init__(self, ttl_seconds: int = 300):
        self.ttl_seconds = ttl_seconds
        self.lock = threading.Lock()
        self.in_flight: Dict[str, Future] = {}
        self.results: Dict[str, tuple[bool, Any, float]] = {}
    
    def get_or_execute(self, request_id: str, fn: Callable[[], Any]) -> Any:
        if not request_id:
            raise ValueError("request_id required")
        
        # 快速路径：检查已缓存且未过期的结果
        entry = self.results.get(request_id)
        if entry is not None:
            success, value, expire_at = entry
            if expire_at > time.time():
                if success:
                    return value
                else:
                    raise value
        
        with self.lock:
            # 双重检查：可能在等待锁期间结果已被其他线程写入
            entry = self.results.get(request_id)
            if entry is not None:
                success, value, expire_at = entry
                if expire_at > time.time():
                    if success:
                        return value
                    else:
                        raise value
            
            if request_id in self.in_flight:
                future = self.in_flight[request_id]
            else:
                future = Future()
                self.in_flight[request_id] = future
        
        if future != self.in_flight.get(request_id):
            return future.result()
        
        try:
            result = fn()
            self.results[request_id] = (True, result, time.time() + self.ttl_seconds)
            future.set_result(result)
            return result
        except Exception as e:
            self.results[request_id] = (False, e, time.time() + 60)
            future.set_exception(e)
            raise
        finally:
            with self.lock:
                del self.in_flight[request_id]
```

**伪代码参考（Go 风格）**

```go
import (
    "sync"
    "time"
)

type cacheEntry struct {
    success  bool
    value    interface{}
    expireAt time.Time
}

type IdempotentCache struct {
    ttlSeconds int
    mu         sync.Mutex
    inFlight   map[string]*future
    results    map[string]cacheEntry
}

func NewIdempotentCache(ttlSeconds int) *IdempotentCache {
    return &IdempotentCache{
        ttlSeconds: ttlSeconds,
        inFlight:   make(map[string]*future),
        results:    make(map[string]cacheEntry),
    }
}

func (c *IdempotentCache) GetOrExecute(requestID string, fn func() (interface{}, error)) (interface{}, error) {
    if requestID == "" {
        return nil, errors.New("request_id required")
    }
    
    // 快速路径
    if entry, ok := c.results[requestID]; ok && time.Now().Before(entry.expireAt) {
        if entry.success {
            return entry.value, nil
        }
        return nil, entry.value.(error)
    }
    
    c.mu.Lock()
    // 双重检查
    if entry, ok := c.results[requestID]; ok && time.Now().Before(entry.expireAt) {
        c.mu.Unlock()
        if entry.success {
            return entry.value, nil
        }
        return nil, entry.value.(error)
    }
    
    if f, ok := c.inFlight[requestID]; ok {
        c.mu.Unlock()
        return f.wait()
    }
    
    f := newFuture()
    c.inFlight[requestID] = f
    c.mu.Unlock()
    
    result, err := fn()
    var entry cacheEntry
    if err != nil {
        entry = cacheEntry{success: false, value: err, expireAt: time.Now().Add(60 * time.Second)}
        f.setError(err)
    } else {
        entry = cacheEntry{success: true, value: result, expireAt: time.Now().Add(time.Duration(c.ttlSeconds) * time.Second)}
        f.setResult(result)
    }
    
    c.mu.Lock()
    c.results[requestID] = entry
    delete(c.inFlight, requestID)
    c.mu.Unlock()
    
    return result, err
}
```

### 题 2：ReAct 循环与 Tool Calls 执行器

**场景**  
需要实现 Agent 核心运行时：接收用户输入后进入 ReAct 循环，调用 LLM 并解析其返回的 JSON（可能是 `final_answer` 或 `tool_call`）。执行工具后将结果作为 `observation` 注入消息历史，继续循环，最多执行 5 步。

**要求**  
写出 ReAct 循环的主干逻辑以及 LLM 响应解析和分发的代码结构。

**考察点**
- 循环控制与状态维护：正确维护 `messages` 列表、最大步数限制、退出条件
- LLM 响应解析与分支：安全解析 JSON、区分响应类型、处理解析失败
- Observation 注入：以正确的消息角色追加工具执行结果
- 错误处理与边界：工具不存在、执行失败、达到最大步数时的处理

**分级评估**

| 等级 | 表现 |
| :--- | :--- |
| L4 | 完整写出循环、解析、分支、错误处理；讨论工具调用消息格式（含 `tool_call_id`）；解析失败时有限重试；工具结果截断策略。 |
| L3 | 写出核心循环和分支，正确处理 `final_answer` 和 `tool_call`；考虑了 JSON 解析失败和工具执行失败；工具分发使用字典映射。 |
| L2 | 能写出基本循环和解析思路，但忽略解析失败的重试或工具执行失败时直接崩溃。经提示后能补充。 |
| L1 | 无法写出清晰的循环结构；无法区分不同响应类型；逻辑混乱或存在死循环风险。 |

**关于消息格式的说明**

本题不强制要求特定 LLM 提供商的消息格式。候选人可使用 OpenAI 风格的 `tool_calls`、Anthropic 的 XML 风格，或自定义 JSON 结构。评分时关注以下要点：
- 工具调用信息（名称、参数）是否被正确传递
- 工具执行结果是否作为独立消息追加到历史中
- 消息角色是否能区分用户、助手、系统、工具结果

只要上述要点清晰，具体字段命名不影响评分。

### 题 3：安全的资源释放

**场景**  
编写一个函数，需要先获取分布式锁，再打开日志文件，然后执行一段可能抛出异常的业务逻辑。无论业务逻辑成功或失败，锁必须被释放，文件必须被关闭。

**要求**  
写出函数的主干结构，确保资源在任何退出路径下都被正确释放。

**考察点**
- 语言级别的资源管理机制（`defer`、`try-finally`、`with` 上下文管理器）
- 释放顺序（逆序或按依赖顺序）
- 释放操作本身的异常处理（不掩盖业务异常）

**分级评估**

| 等级 | 表现 |
| :--- | :--- |

**分级评估**

**设计说明**

本题展示的是**乐观锁**方案：通过 `version` 字段在更新时检测并发冲突，不加行锁。生产环境中也可使用 `SELECT ... FOR UPDATE` 的**悲观锁**方案（事务期间持有行锁）。两者适用场景不同：
- 乐观锁：冲突率低、事务短
- 悲观锁：冲突率高、需严格串行化

候选人若提及悲观锁作为替代方案，属于加分表现。

**伪代码参考（Python 风格）**

```python
import json
import random
import time

class SessionStore:
    def __init__(self, db):
        self.db = db
        self.max_retries = 3
    
    def update_session(self, session_id: str, update_fn):
        for retry in range(self.max_retries + 1):
            tx = self.db.begin_transaction()
            try:
                row = tx.query_one(
                    "SELECT data, version FROM sessions WHERE id = ?",
                    (session_id,)
                )
                if row is None:
                    raise ValueError("session not found")
                
                current_data = json.loads(row["data"])
                current_version = row["version"]
                new_data = update_fn(current_data)
                
                result = tx.execute(
                    "UPDATE sessions SET data = ?, version = version + 1 WHERE id = ? AND version = ?",
                    (json.dumps(new_data), session_id, current_version)
                )
                
                if result.affected_rows == 0:
                    tx.rollback()
                    if retry == self.max_retries:
                        raise RuntimeError("update conflict, max retries exceeded")
                    time.sleep(random.uniform(0.01, 0.05))
                    continue
                
                tx.commit()
                return new_data
            except Exception:
                tx.rollback()
                raise
```

```go
import (
    "encoding/json"
    "math/rand"
    "time"
)

type SessionStore struct {
    db         *sql.DB
    maxRetries int
}

func NewSessionStore(db *sql.DB) *SessionStore {
    return &SessionStore{db: db, maxRetries: 3}
}

func (s *SessionStore) UpdateSession(sessionID string, updateFn func(map[string]interface{}) map[string]interface{}) (map[string]interface{}, error) {
    for retry := 0; retry <= s.maxRetries; retry++ {
        tx, err := s.db.Begin()
        if err != nil {
            return nil, err
        }
        
        var dataStr string
        var version int
        err = tx.QueryRow(
            "SELECT data, version FROM sessions WHERE id = ?",
            sessionID,
        ).Scan(&dataStr, &version)
        if err != nil {
            tx.Rollback()
            return nil, err
        }
        
        var currentData map[string]interface{}
        if err := json.Unmarshal([]byte(dataStr), &currentData); err != nil {
            tx.Rollback()
            return nil, err
        }
        
        newData := updateFn(currentData)
        newDataBytes, _ := json.Marshal(newData)
        
        result, err := tx.Exec(
            "UPDATE sessions SET data = ?, version = version + 1 WHERE id = ? AND version = ?",
            string(newDataBytes), sessionID, version,
        )
        if err != nil {
            tx.Rollback()
            return nil, err
        }
        
        affected, _ := result.RowsAffected()
        if affected == 0 {
            tx.Rollback()
            if retry == s.maxRetries {
                return nil, errors.New("update conflict, max retries exceeded")
            }
            time.Sleep(time.Duration(rand.Intn(40)+10) * time.Millisecond)
            continue
        }
        
        if err := tx.Commit(); err != nil {
            return nil, err
        }
        return newData, nil
    }
    return nil, nil
}
```

```python
import time
import threading
from collections import deque
from typing import Dict

class RateLimiter:
    def __init__(self, window_sec: int = 60, max_requests: int = 10):
        self.window_sec = window_sec
        self.max_requests = max_requests
        self.lock = threading.Lock()
        self.records: Dict[str, deque] = {}
    
    def allow(self, user_id: str) -> bool:
        with self.lock:
            if user_id not in self.records:
                self.records[user_id] = deque()
            queue = self.records[user_id]
            
            now = time.time()
            while queue and queue[0] <= now - self.window_sec:
                queue.popleft()
            
            if len(queue) < self.max_requests:
                queue.append(now)
                return True
            return False
```

```go
    defer r.mu.Unlock()

    if !ok {
        queue = []int64{}
    }

    for i, ts := range queue {
        if ts > cutoff {
            validIdx = i
            break
        }
    }
    queue = queue[validIdx:]

        r.records[userID] = append(queue, now)
        return true
    }
    r.records[userID] = queue
    return false
}
```

```python
import heapq
import threading
import time
from dataclasses import dataclass
from typing import Callable, Dict, Optional

@dataclass(order=True)
class DelayedEntry:
    execute_at: float
    task_id: str
    task_fn: Callable = field(compare=False)
    cancelled: bool = field(default=False, compare=False)

class DelayedTaskExecutor:
    def __init__(self):
        self.lock = threading.Lock()
        self.cond = threading.Condition(self.lock)
        self.heap: List[DelayedEntry] = []
        self.task_map: Dict[str, DelayedEntry] = {}
        self.running = True
    
    def schedule(self, task_id: str, task_fn: Callable, delay_seconds: float):
        with self.lock:
            execute_at = time.time() + delay_seconds
            entry = DelayedEntry(execute_at, task_id, task_fn)
            heapq.heappush(self.heap, entry)
            self.task_map[task_id] = entry
            self.cond.notify()
    
    def cancel(self, task_id: str):
        with self.lock:
            if task_id in self.task_map:
                self.task_map[task_id].cancelled = True
    
    def run(self):
        while self.running:
            with self.lock:
                while not self.heap:
                    self.cond.wait()
                    if not self.running:
                        return
                
                entry = self.heap[0]
                now = time.time()
                if entry.execute_at > now:
                    self.cond.wait(timeout=entry.execute_at - now)
                    continue
                
                heapq.heappop(self.heap)
                self.task_map.pop(entry.task_id, None)
            
            if not entry.cancelled:
                try:
                    entry.task_fn()
                except Exception as e:
                    log.error("delayed task failed", {"task_id": entry.task_id, "error": e})
    
    def shutdown(self):
        with self.lock:
            self.running = False
            self.cond.notify_all()
```

```go
)

    TaskID    string

    Cancelled bool
    index     int

    taskMap  map[string]*DelayedEntry

        running: true,
    }
    e.cond = sync.NewCond(&e.mu)

}

func (e *DelayedTaskExecutor) Schedule(taskID string, taskFn func(), delaySeconds float64) {
    e.mu.Lock()
    defer e.mu.Unlock()
    
    entry := &DelayedEntry{
        ExecuteAt: time.Now().Add(time.Duration(delaySeconds * float64(time.Second))),
        TaskID:    taskID,
        TaskFn:    taskFn,
    }
    heap.Push(&e.heap, entry)
    e.taskMap[taskID] = entry
    e.cond.Signal()
}

func (e *DelayedTaskExecutor) Cancel(taskID string) {
    e.mu.Lock()
    defer e.mu.Unlock()
    if entry, ok := e.taskMap[taskID]; ok {
        entry.Cancelled = true
    }
}

func (e *DelayedTaskExecutor) Run() {
    for e.running {
        e.mu.Lock()
        for e.heap.Len() == 0 && e.running {
            e.cond.Wait()
        }
        if !e.running {
            e.mu.Unlock()
            return
        }
        
        entry := e.heap[0]
        now := time.Now()
        if entry.ExecuteAt.After(now) {
            e.cond.Wait(entry.ExecuteAt.Sub(now))
            e.mu.Unlock()
            continue
        }
        
        heap.Pop(&e.heap)
        delete(e.taskMap, entry.TaskID)
        e.mu.Unlock()
        
        if !entry.Cancelled {
            func() {
                defer func() {
                    if r := recover(); r != nil {
                        log.Error("delayed task panicked", "task_id", entry.TaskID, "panic", r)
                    }
                }()
                entry.TaskFn()
            }()
        }
    }
}

func (e *DelayedTaskExecutor) Shutdown() {
    e.mu.Lock()
    defer e.mu.Unlock()
    e.running = false
    e.cond.Broadcast()
}
```

```python
import subprocess
import resource
from typing import List, Dict, Any

ALLOWED_COMMANDS = {"python3", "node", "ffmpeg", "git"}
FORBIDDEN_PATTERNS = ["|", ";", "&&", "||", ">", "<", "$(", "`"]

def safe_execute_command(command_name: str, args: List[str]) -> Dict[str, Any]:
    if command_name not in ALLOWED_COMMANDS:
        raise ValueError(f"Command not allowed: {command_name}")
    
    for arg in args:
        for pattern in FORBIDDEN_PATTERNS:
            if pattern in arg:
                raise ValueError(f"Forbidden pattern in argument: {pattern}")
    
    if len(args) > 10:
        raise ValueError("Too many arguments")
    for arg in args:
        if len(arg) > 1024:
            raise ValueError("Argument too long")
    
    try:
        result = subprocess.run(
            [command_name] + args,
            capture_output=True,
            timeout=5,
            text=True,
            preexec_fn=lambda: (
                resource.setrlimit(resource.RLIMIT_CPU, (2, 2)),
                resource.setrlimit(resource.RLIMIT_AS, (512 * 1024 * 1024, 512 * 1024 * 1024))
            )
        )
        if result.returncode != 0:
            return {"success": False, "error": result.stderr}
        return {"success": True, "output": result.stdout}
    except subprocess.TimeoutExpired:
        return {"success": False, "error": "Command execution timeout"}
    except Exception as e:
        return {"success": False, "error": str(e)}
```

**伪代码参考（Go 风格）**

```go
import (
    "context"
    "os/exec"
    "strings"
    "time"
)

var allowedCommands = map[string]bool{
    "python3": true,
    "node":    true,
    "ffmpeg":  true,
    "git":     true,
}

var forbiddenPatterns = []string{"|", ";", "&&", "||", ">", "<", "$(", "`"}

func safeExecuteCommand(commandName string, args []string) map[string]interface{} {
    if !allowedCommands[commandName] {
        return map[string]interface{}{"success": false, "error": "Command not allowed: " + commandName}
    }
    
    for _, arg := range args {
        for _, pattern := range forbiddenPatterns {
            if strings.Contains(arg, pattern) {
                return map[string]interface{}{"success": false, "error": "Forbidden pattern in argument: " + pattern}
            }
        }
    }
    
    if len(args) > 10 {
        return map[string]interface{}{"success": false, "error": "Too many arguments"}
    }
    for _, arg := range args {
        if len(arg) > 1024 {
            return map[string]interface{}{"success": false, "error": "Argument too long"}
        }
    }
    
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()
    
    cmd := exec.CommandContext(ctx, commandName, args...)
    output, err := cmd.CombinedOutput()
    
    if ctx.Err() == context.DeadlineExceeded {
        return map[string]interface{}{"success": false, "error": "Command execution timeout"}
    }
    if err != nil {
        return map[string]interface{}{"success": false, "error": err.Error()}
    }
    return map[string]interface{}{"success": true, "output": string(output)}
}
```
